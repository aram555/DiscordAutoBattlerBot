**WebBattler** — это админ-панель для ведения пошаговой стратегической сессии.
Через веб-интерфейс можно управлять игровым миром: странами, провинциями, городами, армиями, юнитами и производством.

## Для кого эта программа

Проект рассчитан в первую очередь на:
- ведущего/администратора игровой сессии;
- команду, которая настраивает игровой мир и продвигает ходы;
- пользователей, которым нужен визуальный интерфейс вместо ручного редактирования данных.

## Что делает WebBattler

После запуска в панели доступны основные сценарии:

1. **Авторизация администратора**
   - регистрация и вход в систему;
   - работа только под учётной записью администратора.

2. **Управление сессией**
   - создание игровых сессий;
   - активация/остановка сессии;
   - завершение хода (turn).

3. **Управление картой и сущностями**
   - страны, провинции, города;
   - армии и юниты;
   - шаблоны юнитов и зданий;

4. **Редактирование данных**
   - отдельная страница редактирования текущей сессии;
   - изменение уже созданных сущностей без ручной работы с БД.

## Текущие достоинства проекта

- **Понятный веб-интерфейс** для администрирования сессии.
- **Разделение по слоям**: Web / Services / DAL — проще сопровождать и развивать.
- **Turn-based логика** уже встроена: можно двигать ход и обрабатывать производственные задачи.
- **Интеграция с Discord**: проект готов к работе с ботом через токен и команды.
- **Миграции EF Core**: база данных разворачивается через стандартный workflow.

## Как запустить программу

### 1) Что нужно установить

- .NET SDK 8.0+
- SQL Server LocalDB (или любой доступный SQL Server)

### 2) Склонировать проект

```bash
git clone <repo-url>
cd WebBattler
```

### 3) Восстановить пакеты и собрать решение

```bash
dotnet restore
dotnet build WebBattler.sln
```

### 4) Применить миграции базы

```bash
dotnet ef database update --project WebBattler.DAL --startup-project WebBattler
```

Если команда `dotnet ef` не установлена:

```bash
dotnet tool install --global dotnet-ef
```

### 5) Указать токен Discord-бота

В репозитории токен хранить не нужно и небезопасно.
Для запуска укажи свой токен через user-secrets:

```bash
dotnet user-secrets init --project WebBattler
dotnet user-secrets set "Discord:Token" "<your-token>" --project WebBattler
```

> В текущей реализации сервис Discord-бота подключается при старте приложения,
> поэтому без значения `Discord:Token` запуск может завершиться ошибкой авторизации.

### 6) Запустить приложение

```bash
dotnet run --project WebBattler
```

После запуска открой адрес из консоли (обычно `https://localhost:<port>`).

## Техническая структура (кратко)

```text
WebBattler.sln
├── WebBattler/               # Веб-приложение (контроллеры, views, запуск)
├── WebBattler.Services/      # Бизнес-логика
└── WebBattler.DAL/           # БД, сущности, репозитории, миграции
```
